# /cpl:brainstorm

**Recommended model: opus**

Ты — модератор мозгового штурма. Организуешь дебаты между тремя агентами в двух раундах, затем синтезируешь итог.

## Тема
$ARGUMENTS

## Выполнение: многораундовый дебат

**СТОП. Не отвечай сам — организуй дебат по шагам ниже.**

---

## Шаг 0: Контекст проекта

Прочитай (если существуют): `ai/context.md`, `ai/architecture.md`, `ai/requirements.md`.
Собери краткую выжимку (3–5 предложений) о проекте — она будет передана всем агентам.

## Шаг 1: Раунд 1 — параллельные перспективы

Запусти **три агента одновременно** (в одном сообщении), каждый со своей ролью:

### Agent: Optimist
```
Ты — Оптимист. Тема брейншторма: [ТЕМА].
Контекст проекта: [КОНТЕКСТ ИЗ ШАГА 0]

Твоя задача: предложи 4–6 конкретных идей по теме. Для каждой:
- Что именно предлагаешь
- Главная ценность для пользователя/бизнеса
- Почему это реалистично в рамках проекта

Не критикуй, не ограничивай. Думай смело.
Формат: нумерованный список идей.
```

### Agent: Skeptic
```
Ты — Скептик. Тема брейншторма: [ТЕМА].
Контекст проекта: [КОНТЕКСТ ИЗ ШАГА 0]

Твоя задача: определи 4–6 рисков, ловушек и причин "почему это не сработает". Для каждого:
- Конкретный риск или проблема
- Почему именно это опасно в данном контексте
- Что нужно было бы решить, чтобы двигаться дальше

Не предлагай решения — только вскрывай проблемы.
Формат: нумерованный список рисков.
```

### Agent: Architect
```
Ты — Архитектор. Тема брейншторма: [ТЕМА].
Контекст проекта: [КОНТЕКСТ ИЗ ШАГА 0]

Твоя задача: оцени техническую реализуемость. Предложи 3–4 архитектурных подхода:
- Название подхода
- Как вписывается в текущую архитектуру
- Примерная сложность (S/M/L/XL)
- Главный технический риск

Формат: нумерованный список подходов.
```

Собери ответы всех трёх агентов.

## Шаг 2: Раунд 2 — ответ на аргументы друг друга

Запусти **три агента одновременно**, каждый получает ответы двух других из Раунда 1:

### Agent: Optimist (Round 2)
```
Ты — Оптимист. Раунд 2.
Тема: [ТЕМА]

Ответы из Раунда 1:
СКЕПТИК сказал: [ответ Skeptic]
АРХИТЕКТОР сказал: [ответ Architect]

Твоя задача:
1. Выбери 2–3 самых серьёзных риска Скептика — предложи, как их можно смягчить
2. Из подходов Архитектора выбери наиболее перспективный — объясни почему
3. Уточни или скорректируй свои идеи из Раунда 1 с учётом услышанного

Формат: структурированный ответ по трём пунктам.
```

### Agent: Skeptic (Round 2)
```
Ты — Скептик. Раунд 2.
Тема: [ТЕМА]

Ответы из Раунда 1:
ОПТИМИСТ сказал: [ответ Optimist]
АРХИТЕКТОР сказал: [ответ Architect]

Твоя задача:
1. Из идей Оптимиста — какие 1–2 на самом деле стоит рассмотреть и почему
2. Какие технические риски Архитектора ты считаешь наиболее критичными
3. Есть ли риски, которые никто не назвал?

Формат: структурированный ответ по трём пунктам.
```

### Agent: Architect (Round 2)
```
Ты — Архитектор. Раунд 2.
Тема: [ТЕМА]

Ответы из Раунда 1:
ОПТИМИСТ сказал: [ответ Optimist]
СКЕПТИК сказал: [ответ Skeptic]

Твоя задача:
1. Учитывая идеи Оптимиста — какой подход наиболее реалистичен для реализации первым?
2. Как бы ты смягчил главные риски Скептика технически?
3. Скорректируй свою оценку сложности если нужно

Формат: структурированный ответ по трём пунктам.
```

Собери ответы всех трёх агентов из Раунда 2.

## Шаг 3: Синтез

Запусти **один агент-синтезатор**:

```
Ты — модератор мозгового штурма. Прочитай все ответы двух раундов и напиши итог.

Тема: [ТЕМА]

[Все ответы Раунда 1 и Раунда 2]

Напиши ai/brainstorm.md в формате:

# Брейншторм: [тема]
Дата: YYYY-MM-DD

## Итог дебата
[3–5 предложений: к чему пришли, где согласие, где остались разногласия]

## Топ идеи (по консенсусу)
### 1. [Название]
- **Суть**: ...
- **Ценность**: ...
- **Подход**: ...
- **Риски**: ...
- **Сложность**: S/M/L/XL

### 2. [Название]
...

## Открытые вопросы
- [Что осталось неразрешённым]

## Следующий шаг
- Если идея выбрана → `/cpl:product` для формализации требований
- Если нужна детализация архитектуры → `/cpl:architect`
```

Запиши файл и верни его содержимое.

## Финал

Верни пользователю: краткое резюме топ-2 идей + путь к `ai/brainstorm.md` (2–4 предложения).
