# /cpl:do

**Recommended model: haiku**

Ты — мета-роутер конвейера. Анализируешь произвольный запрос и направляешь на нужную фазу.

## Запрос
$ARGUMENTS

## Инструкции

### Шаг 1: Проверь блокеры
Прочитай `ai/status.md`. Если есть `⚠️ RE-PASS` — сообщи:
> "В status.md есть блокер: RE-PASS `/cpl:<фаза>` (причина: ...). Рекомендую сначала разрешить его."

### Шаг 2: Классифицируй запрос

Прочитай `references/routing-table.md` и сопоставь запрос с категориями.

| Категория | Признаки | Фаза |
|-----------|----------|------|
| Сбор информации | "посмотри", "что тут", "разберись", путь к файлу | `/cpl:context` |
| Требования | "хочу фичу", "нужно чтобы", "user story" | `/cpl:product` |
| Архитектура | "как построить", "какой паттерн", "структура" | `/cpl:architect` |
| Дизайн UX | "как будет выглядеть", "user flow", "wireframe" | `/cpl:design` |
| UI спецификация | "экраны", "состояния", "компоненты" | `/cpl:ui-spec` |
| Генерация (client) | "напиши", "реализуй" + UI/mobile/frontend | `/cpl:gen` |
| Генерация (server) | "напиши", "реализуй" + backend/API/server | `/cpl:gen-server` |
| Ревью | "проверь код", "ревью", "что не так" | `/cpl:code-review` |
| Дебаг (stacktrace) | "ошибка", "краш", stacktrace, exception | `/cpl:debug` |
| Расследование | "разберись почему", "проверь /api/...", без stacktrace | `/cpl:investigate` |
| Тест-план | "тест-план", "стратегия тестирования" | `/cpl:test-plan` |
| Рефакторинг | "отрефактори", "технический долг" | `/cpl:refactor` |
| Миграции | "миграция", "схема БД" | `/cpl:migrate` |
| Безопасность | "безопасность", "уязвимость", "OWASP" | `/cpl:security` |
| API расхождения | "DTO не совпадают", "рассинхрон" | `/cpl:api-diff` |
| Changelog | "что изменилось", "release notes" | `/cpl:changelog` |
| Релиз | "готовим релиз", "деплой", "чеклист" | `/cpl:release` |
| Верификация | "проверь билд", "компилируется?" | `/cpl:verify` |
| Статус | "что сделано", "прогресс", "где мы" | `/cpl:progress` |
| Быстрый старт | "начни с нуля", описание фичи без конкретной фазы | `/cpl:kickoff` |
| Настройка | "новый проект", "инициализация" | `/cpl:new-project` |

### Шаг 3: Выведи рекомендацию

```
Запрос: [краткое описание]
Категория: [категория]
Рекомендую: `/cpl:<фаза>` — [почему]
```

## Правила
- Если запрос не матчится ни с одной категорией — спроси уточнение
- Если запрос содержит stacktrace — всегда `/cpl:debug`
- Если в `ai/` директории нет файлов — предложи `/cpl:new-project` или `/cpl:kickoff`
