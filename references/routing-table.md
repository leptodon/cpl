## Автоопределение следующей фазы

Если пользователь НЕ указал конкретную задачу или спрашивает "что дальше":

### Приоритет 1: Блокеры (RE-PASS)
Если в `ai/status.md` есть `⚠️ RE-PASS` — предложи указанную фазу:
> "В status.md есть блокер: RE-PASS `/cpi:architect` (причина: ...). Рекомендую начать с него."

### Приоритет 2: Прогресс артефактов
Определи первый незаполненный артефакт в цепочке и предложи соответствующую фазу:

```
context [ ] → /cpi:gather
requirements [ ] → /cpi:product
architecture [ ] → /cpi:architect
ui_spec [ ] → /cpi:design + /cpi:ui-spec
test_plan [ ] → /cpi:test-plan
tasks [x] + implementation [ ] → /cpi:gen (или /cpi:gen-server)
implementation [x] + review [ ] → /cpi:code-review
всё [x] → /cpi:release (или /cpi:changelog)
```

### Приоритет 3: Next action из status.md
Если в `ai/status.md` указан "Next action" — предложи его.

### Формат предложения
```
Текущая фаза: [фаза из status.md]
Прогресс: N/M артефактов заполнено
Рекомендую: `/фаза` — [почему]
```
