# /cpl:pause

**Recommended model: haiku**

Сохранение позиции и контекста для продолжения в следующей сессии.

## Контекст
$ARGUMENTS

## Инструкции

### Шаг 1: Определи текущую фазу
Прочитай `ai/status.md` — возьми текущую фазу и прогресс.

### Шаг 2: Собери состояние

1. **Выполненные таски**: прочитай `ai/tasks.md`, собери все `[x]`
2. **Оставшиеся таски**: собери все `[ ]`
3. **Блокеры**: из `ai/status.md` и `ai/debug.md` (если есть)
4. **Незакоммиченные файлы**: выполни `git status`
5. **Ключевые решения**: из `ai/architecture.md` (ADR), `ai/impl_notes.md`

### Шаг 3: Запиши handoff

Создай/обнови `ai/handoff.md`:

```markdown
# Handoff
Дата: YYYY-MM-DD

## Позиция
- Фаза: [текущая]
- Последний выполненный таск: [номер и описание]
- Прогресс: N/M тасков

## Выполнено
- [x] Task 1: описание
- [x] Task 2: описание

## Осталось
- [ ] Task 3: описание
- [ ] Task 4: описание

## Блокеры
- [описание блокера, если есть]

## Незакоммиченные файлы
- [список файлов из git status, если есть]

## Ключевые решения
- ADR-1: [решение]
- [другие решения из сессии]

## Контекст от пользователя
[если пользователь передал $ARGUMENTS — записать здесь]

## Следующее действие
- Команда: `/cpl:<фаза>`
- Что делать: [конкретное описание]
```

### Шаг 4: Обнови status.md
Добавь в `ai/status.md`:
```
## Пауза
- Дата: YYYY-MM-DD
- Позиция: [фаза], таск N/M
- Продолжение: `/cpl:resume`
```

### Шаг 5: Подтверди

```
Сессия сохранена.
- Фаза: [фаза]
- Прогресс: N/M
- Продолжение: `/cpl:resume`
```

## Правила
- НЕ коммить незавершённую работу без согласия пользователя
- Если есть незакоммиченные файлы — предупреди и спроси нужно ли закоммитить
