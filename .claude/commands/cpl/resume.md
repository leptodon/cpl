# /cpl:resume

**Recommended model: haiku**

Восстановление контекста из предыдущей сессии.

## Инструкции

### Шаг 1: Загрузи контекст
Прочитай:
1. `ai/handoff.md` — основной источник (позиция, таски, решения)
2. `ai/status.md` — общий прогресс и блокеры

Если `ai/handoff.md` не существует — сообщи:
> "Нет сохранённой сессии. Используй `/cpl:progress` для обзора текущего состояния или `/cpl:next` для определения следующего шага."

### Шаг 2: Проверь актуальность
1. Проверь что указанные коммиты/файлы существуют (`git log --oneline -5`)
2. Проверь что незакоммиченные файлы из handoff всё ещё присутствуют (`git status`)
3. Если что-то изменилось — отметь расхождения

### Шаг 3: Выведи сводку

```
## Восстановление сессии

Фаза: [фаза]
Прогресс: N/M тасков

### Выполнено
- Task 1: описание
- Task 2: описание

### Осталось
- Task 3: описание
- Task 4: описание

### Ключевые решения (из прошлой сессии)
- [решение 1]
- [решение 2]

### Блокеры
- [если есть]

### Расхождения с handoff
- [если файлы изменились с момента паузы]

## Рекомендую
`/cpl:<фаза>` — [конкретное действие]
```

### Шаг 4: Очистка (опционально)
Если пользователь продолжает работу — предложи:
> "Удалить `ai/handoff.md` и пометку паузы в `ai/status.md`? Или оставить до завершения фазы?"

## Правила
- НЕ менять код — только чтение и восстановление контекста
- Если обнаружены расхождения — явно указать и спросить как поступить
- Если handoff.md устарел (>7 дней) — предупредить что контекст может быть неактуален
