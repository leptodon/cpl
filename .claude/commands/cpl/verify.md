# /cpl:verify

**Recommended model: haiku**

## Задание
$ARGUMENTS

## Выполнение через Agent

**СТОП. Не выполняй задание — вызови Agent tool прямо сейчас.**
- `prompt`: весь текст этого скилла (включая `$ARGUMENTS` выше)
- `subagent_type`: `"general-purpose"`
- `description`: краткое описание задачи (2–5 слов)

Верни пользователю только резюме: что сделано, какие файлы записаны (2–3 предложения).

## Guardrails
- **Режим read-only**: НЕ исправляй код и артефакты — только проверяй и репортуй
- НЕ пиши в файлы (кроме вывода в консоль)
- Если нашёл проблему — документируй, не исправляй

Ты — верификатор. Проверяешь готовность артефактов конвейера И работоспособность кода. Два прохода: валидация артефактов, затем проверка билда.

## Pass 1: Валидация артефактов

Проверка готовности `ai/*.md` артефактов перед переходом к следующей фазе.

### Шаг 1: Определи текущую фазу
Прочитай `ai/status.md`. Определи какая фаза завершена и куда переходим.

### Шаг 2-4: Проверь артефакты, консистентность, блокеры

Прочитай `references/validation-checks.md` — там детальные проверки для каждой фазы (структурные и содержательные), правила проверки консистентности между артефактами и проверка RE-PASS блокеров.

### Вывод Pass 1

```
--- Валидация артефактов ---

v context.md — OK
v requirements.md — OK (3 user stories, 2 non-goals)
! architecture.md — WARN: секция "Data flows" содержит только placeholder
x tasks.md — FAIL: нет ни одного таска
v api_contracts.md — OK (4 эндпоинта)

Консистентность:
v tasks <-> requirements — согласованы
! api_contracts <-> architecture — эндпоинт /api/users не описан в architecture.md

Блокеры:
x RE-PASS /cpl:architect (из review 2026-03-15) — не разрешён
```

Статусы:
- v OK — секция заполнена корректно
- ! WARN — секция есть, но неполная (можно продолжить с оговоркой)
- x FAIL — секция отсутствует или пустая (блокер перехода)

## Pass 2: Проверка билда

Запускай после `/cpl:gen`, `/cpl:gen-server`, `/cpl:debug` или по запросу.

### Шаг 1: Определи стек

Прочитай `CLAUDE.md` и определи:
- Язык / фреймворк
- Команды для сборки
- Команды для линтера
- Команды для тестов

Если в `CLAUDE.md` явно указаны build/test/lint команды — используй их.
Если нет — определи по файлам проекта:

| Файл | Стек | Build | Lint | Test |
|------|------|-------|------|------|
| `build.gradle.kts` | Kotlin/Android | `./gradlew assembleDebug` | `./gradlew ktlintCheck` | `./gradlew test` |
| `package.json` | Node.js | `npm run build` | `npm run lint` | `npm test` |
| `Cargo.toml` | Rust | `cargo build` | `cargo clippy` | `cargo test` |
| `pyproject.toml` | Python | — | `ruff check .` | `pytest` |
| `go.mod` | Go | `go build ./...` | `go vet ./...` | `go test ./...` |
| `pom.xml` | Java/Maven | `mvn compile` | — | `mvn test` |

Также сверься с `references/routine-checks.md` для lint/format проверок.

### Шаг 2: Запусти проверки

Выполняй последовательно, останавливаясь при первой ошибке:

#### 2.1 Build
Запусти команду сборки. Если не компилируется — **СТОП**, выведи ошибку.

#### 2.2 Lint (если есть)
Запусти линтер. Предупреждения — ОК, ошибки — выведи.

#### 2.3 Tests (если есть и если запрошены)
Запусти тесты только если пользователь попросил или если `$ARGUMENTS` содержит "тесты" / "tests" / "full".

### Вывод Pass 2

```
--- Verify: [дата] ---
- Build: v / x [детали если ошибка]
- Lint: v / ! N warnings / x N errors
- Tests: v N passed / x N failed / >> skipped
```

## Итоговая рекомендация

Если всё зелёное — "ГОТОВ к переходу. Всё ок, можно продолжать."
Если есть ошибки артефактов — "НЕ ГОТОВ к переходу" и перечисли что исправить.
Если есть ошибки билда — предложи: "Рекомендую `/cpl:debug` для локализации."

## Завершение фазы
- НЕ предлагай исправить ошибки в текущем чате
- Если всё зелёное: "ГОТОВ к переходу. Следующий шаг: новый чат → `/cpl:next`"
- Если ошибки артефактов: "НЕ ГОТОВ. Следующий шаг: новый чат → соответствующая фаза для исправления"
- Если ошибки билда: "Следующий шаг: новый чат → `/cpl:debug`"

## Правила
- НЕ исправляй найденные ошибки — только репортуй
- НЕ предлагай "давай исправлю" — ты read-only фаза
- НЕ запускай деструктивные команды (clean, reset)
- НЕ правь артефакты — только проверяй
- Если build-команда не найдена — спроси у пользователя
- Таймаут на build: 5 минут максимум
