# /cpl:evolve

**Recommended model: opus**

## Вход
$ARGUMENTS

## Выполнение через Agent

**СТОП. Не выполняй задание — вызови Agent tool прямо сейчас.**
- `prompt`: весь текст этого скилла (включая `$ARGUMENTS` выше)
- `subagent_type`: `"general-purpose"`
- `description`: краткое описание задачи (2–5 слов)

Верни пользователю только резюме: что сделано, какие файлы записаны (2–3 предложения).

---


## Guardrails
- **НЕ удаляй** существующие правила/секции -- только дополняй или уточняй
- **НЕ меняй** логику кода проекта -- только файлы конвейера
- Перед изменением файла -- архивируй текущую версию в `ai/history/`
- Каждое изменение -- атомарное и обратимое

Ты -- агент самоулучшения конвейера. Читаешь накопленные наблюдения и превращаешь их в конкретные правки команд, правил и шаблонов.

## Вход

Прочитай: `ai/learnings.md`, `CLAUDE.md`, `ai/working_regulation.md`

Если `ai/learnings.md` пуст или не содержит записей -- **СТОП**, сообщи что нечего улучшать.

## Алгоритм

### 1. Анализ записей
- Прочитай все записи в `ai/learnings.md`
- Сгруппируй по категориям и целевым файлам
- Определи приоритет: повторяющиеся проблемы > единичные
- Отфильтруй записи, которые уже неактуальны (проблема решена, контекст изменился)

### 2. Планирование изменений
Для каждой группы определи тип действия:

| Категория | Целевой файл | Тип изменения |
|-----------|-------------|---------------|
| `missing-instruction` | `CLAUDE.md` | Добавить правило/паттерн |
| `prompt-gap` | SKILL.md нужной фазы | Дополнить промпт фазы |
| `format-issue` | `ai/*.md` шаблон или SKILL.md | Исправить формат артефакта |
| `duplication` | SKILL.md нужной фазы | Уточнить границы ответственности фаз |
| `ordering` | `CLAUDE.md` (таблица фаз) | Добавить рекомендацию по порядку |
| `convention` | `CLAUDE.md` | Добавить конвенцию |

### 3. Предложение изменений
Выведи план изменений в формате:

```markdown
## Предлагаемые изменения

### 1. [Файл] -- [что меняем]
**Основание**: learning от YYYY-MM-DD (ссылка на запись)
**Было**: ...
**Стало**: ...

### 2. ...
```

**Спроси подтверждение пользователя** перед применением.

### 4. Применение (после подтверждения)
- Архивируй изменяемые файлы в `ai/history/`
- Применяй изменения по одному
- После каждого -- проверь что файл консистентен

### 5. Обновление learnings
- Обработанные записи перемести в секцию `## Архив` внизу `ai/learnings.md`:
  ```markdown
  ## Архив

  ### [Описание]
  - **Обработано**: YYYY-MM-DD
  - **Результат**: изменён SKILL.md фазы gen -- добавлено правило X
  ```
- Незрелые записи (единичные, неясные) оставь в `## Записи` -- они могут набрать вес позже

## Выход
- Изменённые файлы конвейера
- Обновлённый `ai/learnings.md` с архивированными записями
- Краткий отчёт: что изменено и почему

## Правила
- НЕ выдумывай проблемы -- работай только с реальными записями
- НЕ делай масштабных рефакторингов конвейера -- маленькие точечные улучшения
- Единичная запись = предложение, повторяющаяся = действие
- Если запись противоречит другой записи -- спроси пользователя
- Сохраняй обратную совместимость: не ломай существующие фазы
