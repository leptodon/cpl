# /cpl:debug

**Recommended model: opus**

Ты локализуешь и чинишь баги. По умолчанию — оба шага. Если пользователь явно просит "только посмотри" / "only look" / "только диагностика" — останови после Step A.

## Проблема
$ARGUMENTS

## Выполнение через Agent

**СТОП. Не выполняй задание — вызови Agent tool прямо сейчас.**
- `prompt`: весь текст этого скилла (включая `$ARGUMENTS` выше)
- `subagent_type`: `"general-purpose"`
- `description`: краткое описание задачи (2–5 слов)

Верни пользователю только резюме: что сделано, какие файлы записаны (2–3 предложения).

## Параллельное выполнение гипотез (Step A)

> Эта секция — инструкция для субагента, не для оркестратора.

При 2-3 гипотезах — проверяй параллельно:

1. Выбери 2-3 наиболее вероятных (исключив уже проверенные из `## История попыток` в `ai/debug.md`)
2. Для каждой запусти Agent параллельно: читает конкретные файлы, ищет признаки гипотезы
3. Возвращает: `CONFIRMED (file:line, доказательство)` | `REJECTED (почему)` | `INCONCLUSIVE`
4. По результатам выбери наиболее вероятную для Step B (фикс)

## Step A: Диагностика (read-only)

### Guardrails (Step A)
- **Режим read-only**: НЕ исправляй код — только локализуй проблему
- Только чтение файлов и запись в `ai/debug.md` и `ai/gotchas.md`

### Инструкции
- Проверь `ai/gotchas.md` — возможно, проблема уже документирована
- Проверь `ai/debug.md` — если файл существует, прочитай **## История попыток** и НЕ повторяй уже проверенные гипотезы
1. Прочитай stacktrace/лог/описание
2. Найди релевантные файлы через поиск
3. Составь ранжированный список гипотез (исключи уже проверенные)

### Формат ai/debug.md

```markdown
# Дебаг: [проблема]
Дата: YYYY-MM-DD

## Симптомы
- ...

## Стектрейс / логи
```
(вставка)

### Правила Step A
- НЕ чини баг
- НЕ предлагай рефакторинг
- Только локализация и гипотезы

> Если пользователь попросил "только посмотри" / "only look" / "только диагностика" — **СТОП** здесь. Не переходи к Step B. Завершай: "Диагностика в `ai/debug.md`. Следующий шаг: новый чат → `/cpl:debug` для фикса (Step B)."

## Step B: Фикс

### Guardrails (Step B)
- **Минимальный diff**: меняй только то, что необходимо для фикса
- **НЕ рефактори** окружающий код — только фикс бага
- **НЕ редактируй** файлы в `ai/` кроме `ai/impl_notes.md`, `ai/debug.md` и `ai/gotchas.md`
- Перед коммитом: убедись что проект компилируется

### Вход
Используй гипотезы из Step A (`ai/debug.md`). Если Step A не был выполнен — **СТОП**, выполни сначала Step A.

### Инструкции
1. Прочитай `ai/debug.md` → **## История попыток** — убедись что не повторяешь уже проверенное
2. Возьми наиболее вероятную непроверенную гипотезу
3. Прочитай конкретный файл/метод
4. Пойми корневую причину
5. Сделай **минимальный фикс**
6. Запиши результат в `ai/debug.md` → **## История попыток** (независимо от того, сработало или нет)
7. Если не сработало — вернись к шагу 2 со следующей гипотезой

### Правила Step B
- Минимальный diff — чини только баг
- НЕ рефактори окружающий код
- НЕ добавляй "заодно" улучшения
- Объясни что и почему изменил

## Auto-learnings

После успешного фикса (Step B), автоматически допиши в `ai/gotchas.md`:

```markdown
## [Краткое описание бага]
- **Дата**: YYYY-MM-DD
- **Симптом**: [что наблюдалось]
- **Причина**: [корневая причина]
- **Фикс**: [что изменили]
- **Файл**: [path/file.ext:line]
- **Урок**: [что учитывать в будущем чтобы избежать подобного]
```

Это предотвращает повторение одних и тех же ошибок.

- Если заметил проблему в конвейере (промпт не учёл кейс, не хватает правила, неудобный формат артефакта) — допиши в `ai/learnings.md`

## Антипаттерны
- Повторять уже проверенную гипотезу — всегда читай ## История попыток перед новой итерацией
- Фиксить симптом вместо причины — добавлять null-checks везде без вопроса "почему null?"
- Дебажить без воспроизведения — сначала воспроизведи баг
- Читать только первый фрейм stacktrace — читай полностью
- Стекать несколько гипотез — одна гипотеза за раз
- Бесконечный loop — после 3 провалившихся гипотез останови и эскалируй
- Рефактори "пока здесь" — только минимальный фикс, ничего лишнего
- Спекуляции без доказательств — "скорее всего race condition" без конкретного кода
