# Coding Style Rules

## Иммутабельность

ВСЕГДА создавай новые объекты, НИКОГДА не мутируй:

```javascript
// НЕПРАВИЛЬНО: мутация
function updateUser(user, name) {
  user.name = name  // МУТАЦИЯ!
  return user
}

// ПРАВИЛЬНО: иммутабельность
function updateUser(user, name) {
  return { ...user, name }
}
```

## Организация файлов

МНОГО МАЛЕНЬКИХ ФАЙЛОВ > МАЛО БОЛЬШИХ:
- Высокая связность, низкая зависимость
- 200–400 строк типично, 800 — максимум
- Извлекай утилиты из больших компонентов
- Организация по фичам/доменам, не по типам

## Обработка ошибок

ВСЕГДА обрабатывай ошибки комплексно:

```typescript
try {
  const result = await riskyOperation()
  return result
} catch (error) {
  console.error('Операция провалилась:', error)
  throw new Error('Понятное пользователю сообщение об ошибке')
}
```

## Валидация входных данных

ВСЕГДА валидируй входные данные (на границах системы):

```typescript
import { z } from 'zod'

const schema = z.object({
  email: z.string().email(),
  age: z.number().int().min(0).max(150)
})

const validated = schema.parse(input)
```

## Чеклист качества кода

Перед завершением работы:
- [ ] Код читаем и хорошо именован
- [ ] Функции небольшие (<50 строк)
- [ ] Файлы сфокусированы (<800 строк)
- [ ] Нет глубокой вложенности (>4 уровней)
- [ ] Правильная обработка ошибок
- [ ] Нет console.log в продакшн-коде
- [ ] Нет хардкод-значений
- [ ] Используются иммутабельные паттерны
